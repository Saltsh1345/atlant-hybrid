# Olympic Track — пошаговая настройка ATLANT

Эта система сравнивает вашу технику с **эталоном профи/олимпийцев**, а не с вашими прошлыми тренировками.

---

## Шаг 1. Запуск приложения

1. Откройте терминал в папке `C:\Users\Arman\atlant-hybrid`
2. Если ещё не ставили зависимости: `npm install`
3. Запуск: `npm run dev`
4. В браузере: **http://localhost:3000**

---

## Шаг 2. Ключ Gemini (обязательно для видеоанализа)

1. Скопируйте файл `.env.local.example` → `.env.local` (в корне проекта)
2. Получите ключ: https://aistudio.google.com/apikey
3. В `.env.local` вставьте:
   ```env
   GEMINI_API_KEY=ваш_ключ_сюда
   ```
4. **Не отправляйте ключ в чат** — только в файл на вашем ПК
5. Перезапустите `npm run dev` (Ctrl+C, снова `npm run dev`)

---

## Шаг 3. Калибровка тела (один раз)

1. В приложении откройте **Профиль**
2. Пройдите **Калибровку** — встаньте в полный рост, лицом к камере
3. Дождитесь зелёной отметки «Готово»

Без калибровки углы суставов и 3D-двойник будут менее точными.

---

## Шаг 4. Первая тренировка (бокс или теннис)

1. **Дашборд** → выберите **Бокс** или **Теннис**
2. Нажмите **Тренировка**
3. Разрешите доступ к **камере** (браузер спросит)
4. Дождитесь: «Камера готова» + отсчёт **3-2-1**
5. Слушайте команду (джеб, кросс, форхенд…)
6. На фазе **«Выполняйте удар»** — один резкий удар в камеру
7. Смотрите внизу справа: **«Эталон %»** — это сравнение с профи
8. После 4 команд нажмите **«Анализ и план тренировки»**

### Советы по камере

- Свет спереди, не сзади
- Корпус и руки в кадре (по пояс или в полный рост)
- Телефон/ноутбук на уровне груди, 1.5–2 м от вас

---

## Шаг 5. Разбор после тренировки

На экране **Анализ** вы увидите:

| Блок | Что значит |
|------|------------|
| **Эталон %** | Близость к олимпийской биомеханике |
| **Техника %** | Общая оценка формы камерой |
| **Olympic Track** | Отклонения (скорость, корпус, локоть…) |
| **Фиксация ударов** | Скорость + эталон по каждой команде |
| **Видеоанализ** | Gemini смотрит записанные клипы |
| **Пошаговая настройка** | Чеклист если что-то не работает |

---

## Шаг 6. YOLO на видео (опционально, бесплатно)

Roboflow **не нужен**. Вариант через Colab:

1. Откройте `tools/elite/README.md`
2. Обучите YOLO на публичном датасете (BoxingDataset / boxpunch-detector)
3. Поднимите FastAPI сервер (см. Yolo-App на GitHub)
4. В `.env.local`:
   ```env
   YOLO_PROVIDER=custom
   YOLO_API_URL=http://127.0.0.1:8000/predict
   ```

Без YOLO работают: MediaPipe + эталон + Gemini.

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| Нет видеоанализа | Проверьте `GEMINI_API_KEY` в `.env.local`, перезапуск dev |
| Эталон всегда 0% | Бейте на фазе «Выполняйте», ближе к камере |
| Камера не работает | HTTPS или localhost; разрешение в браузере |
| YOLO не активен | Нормально без ключей — есть Gemini + pose |
